# Управление GitHub Issues

## Когда использовать
- Создание баг-репортов
- Трекинг задач и фич
- Закрытие issues после исправления
- Просмотр текущих задач

## Как работать
1. Определи тип действия (создать/посмотреть/закрыть)
2. Используй GitHub MCP для выполнения
3. Добавь релевантные лейблы и описание

## Примеры

### Создать issue на баг
Используй инструмент `create_issue`:
- title: Краткое описание бага
- body: Подробное описание с шагами воспроизведения
- labels: ["bug"]

Шаблон body:
```
## Описание
[Что происходит]

## Шаги воспроизведения
1. ...
2. ...

## Ожидаемое поведение
[Что должно происходить]

## Фактическое поведение
[Что происходит на самом деле]
```

### Посмотреть открытые issues
Используй инструмент `list_issues` с параметром state: "open"

### Закрыть issue с комментарием
1. Добавь комментарий через `add_issue_comment` с описанием фикса
2. Закрой issue через `update_issue` с state: "closed"

## Важно
- Всегда добавляй лейблы (bug, enhancement, documentation)
- В описании бага указывай шаги воспроизведения
- При закрытии ссылайся на коммит или PR с фиксом
