# Проверка UI через Playwright MCP

## Когда использовать
- Визуальная проверка работы UI
- Прохождение пользовательских сценариев
- Проверка после внесения изменений
- Скриншоты для документации

## Как работать
1. Убедись, что приложение запущено (npm run dev на порту 3000)
2. Используй Playwright MCP для навигации и взаимодействия
3. Делай скриншоты для фиксации результатов

## Примеры сценариев

### Проверка отображения списка задач
1. Открой http://localhost:3000 через `browser_navigate`
2. Сделай скриншот через `browser_screenshot`
3. Проверь, что список задач отображается

### Создание задачи через UI
1. Открой http://localhost:3000
2. Заполни форму создания задачи через `browser_type`
3. Нажми кнопку отправки через `browser_click`
4. Сделай скриншот результата
5. Проверь, что задача появилась в списке

### Проверка фильтрации
1. Открой http://localhost:3000
2. Выбери фильтр по статусу через `browser_click`
3. Сделай скриншот
4. Проверь, что отображаются только задачи с выбранным статусом

## Важно
- Перед проверкой убедись, что dev-сервер запущен
- Делай скриншоты до и после каждого действия
- Описывай что видишь на скриншоте
