# Сквозной сценарий: Полный цикл отладки

## Когда использовать
- Демонстрация совместной работы всех MCP-серверов
- Сквозной сценарий от обнаружения проблемы до её решения

## Сценарий: "Пользователь жалуется, что его задачи пропали"

### Шаг 1: Проверка UI (Playwright MCP)
1. Открой http://localhost:3000
2. Сделай скриншот — видим список задач
3. Проверь, отображаются ли все задачи или есть проблемы

### Шаг 2: Диагностика в БД (PostgreSQL MCP)
1. Проверь количество задач в базе:
```sql
SELECT COUNT(*) FROM "Todo";
```
2. Проверь задачи конкретного пользователя:
```sql
SELECT t.*, u.name FROM "Todo" t
JOIN "User" u ON t."userId" = u.id
WHERE u.name = 'Alice Johnson';
```
3. Найди аномалии:
```sql
SELECT * FROM "Todo" WHERE title = '???' OR description IS NULL;
```

### Шаг 3: Создание issue (GitHub MCP)
1. Создай issue с описанием найденной проблемы
2. Включи данные из диагностики БД
3. Добавь лейбл "bug"

### Шаг 4: Исправление
1. Определи корневую причину проблемы в коде
2. Внеси исправления

### Шаг 5: Проверка исправления (Playwright MCP)
1. Открой приложение снова
2. Сделай скриншот
3. Убедись, что проблема решена

### Шаг 6: Закрытие issue (GitHub MCP)
1. Добавь комментарий к issue с описанием фикса
2. Закрой issue

## Результат
Демонстрация полного цикла: обнаружение → диагностика → трекинг → исправление → верификация → закрытие
